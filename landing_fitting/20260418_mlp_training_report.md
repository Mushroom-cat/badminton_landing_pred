# 20260418 轨迹落点预测对比实验报告

## 实验目标

实验比较两种方法：

1. **MLP 方法**：根据最近20或30个轨迹点的状态预测关键参数 `c1/c2/c3`，再拟合其余参数并计算落点。
2. **无 MLP 方法**：直接从当前可见轨迹拟合全部参数并计算落点。

两种方法使用相同的测试轨迹、轨迹公式、参数约束和落点定义，主要指标为预测落点与真实落点的 XY 平面距离。

## 轨迹公式与预测流程

实验采用论文中的三轴轨迹形式：

```text
x(t) = a1 * exp(b1 * t) + c1
y(t) = a2 * exp(-0.002 * t) + b2 * t + c2
z(t) = a3 * exp(b3 * t) - 0.79 * t + c3
```

其中，`c1/c2/c3` 是三条坐标曲线中的关键偏置参数。MLP 并不直接输出落点，而是根据最近20个可见点提取的位置、速度和加速度，预测这三个参数；随后由当前全部可见点拟合剩余参数。无 MLP 方法则从相同的可见点直接拟合全部参数。

两种方法最后都通过求解 `z(t)=0` 得到落地时间，再代入 `x(t)` 和 `y(t)` 得到预测落点。因此，对比中的输入范围、轨迹方程、落地高度、求根过程和误差定义均保持一致，区别只在于 `c1/c2/c3` 是由 MLP 提供，还是随其他参数一起直接拟合。

## 数据划分与防泄露处理

数据集共有78个文件。划分严格在参数标签质量筛选之前进行，并按原始轨迹组完成：

| 集合       | 文件数 | 说明                                  |
| ---------- | -----: | ------------------------------------- |
| 原始训练组 |     60 | 其中4条参数标签退化，仅从训练组中排除 |
| 实际训练集 |     56 | 478个训练前缀                         |
| 验证集     |      9 | 112个验证前缀                         |
| 测试集     |      9 | 对应8个独立轨迹组，测试集保持完整     |

测试集中有一条轨迹同时存在普通版本和较长的 `_1` 版本。两者始终位于同一集合；统计总体误差时先在该组内平均，再按8个独立轨迹组等权计算，避免同一真实轨迹被重复加权。

审计结果：

- 训练、验证、测试文件名无交集。
- 三个集合的轨迹组无交集。
- 训练与测试之间没有完全相同的轨迹内容。
- 训练与测试之间没有完全相同的真实落点。
- 输入与目标的归一化统计仅由训练集计算。
- 测试落点和测试参数标签不参与模型训练、早停或归一化。

## 20点MLP训练设置

- 输入：位置、速度、加速度，共9维
- 状态窗口：最近20个轨迹点
- 网络：`9 -> 128 -> 128 -> 64 -> 3`
- 输出：`c1/c2/c3`
- 激活函数：ReLU
- Dropout：0.1
- 损失函数：SmoothL1
- 优化器：AdamW
- 初始学习率：`1e-3`
- Weight decay：`1e-5`
- Batch size：64
- 最大训练轮数：300
- Early stopping patience：30
- 随机种子：42
- 实际早停：第123轮

测试前缀上的参数 MAE 为：

| 参数 |      MAE |
| ---- | -------: |
| `c1` |   30.380 |
| `c2` |   40.629 |
| `c3` | 1020.299 |

参数误差不等同于落点误差，因为后续拟合的其他参数可能产生补偿，最终仍以落点误差为主要指标。

## 完整可见轨迹对比

每个测试文件使用其全部可见点预测，并按8个独立轨迹组等权汇总：

| 方法   | Mean（cm） | Median（cm） |  P90（cm） |  Max（cm） |
| ------ | ---------: | -----------: | ---------: | ---------: |
| MLP    |  **7.896** |    **6.953** | **15.440** |     17.833 |
| 无 MLP |     11.288 |       13.250 |     15.648 | **17.659** |

MLP在8个独立轨迹组中的5个上更准确。它降低了平均值和中位数，但并非每条轨迹都优于直接拟合。

## 可见帧逐步缩减实验

从每个测试文件的前51个可见点开始，每次隐藏末尾1个点，直到只剩20点。每个可见帧位置均保留全部测试文件，并按8个独立轨迹组等权汇总。

![可见帧逐步缩减的落点误差对比](20260418_sliding_visibility_comparison.png)

图中实线为组均落点误差，阴影为25%–75%四分位范围，横轴从左到右表示可见帧数由51减少到20。

| 可见帧数 |  MLP（cm） | 无 MLP（cm） |
| -------: | ---------: | -----------: |
|       51 |     12.887 |   **11.448** |
|       45 | **11.857** |       14.837 |
|       40 | **13.045** |       17.445 |
|       35 | **13.455** |       17.654 |
|       30 | **14.891** |       26.897 |
|       25 | **17.103** |       34.829 |
|       20 | **17.416** |       54.195 |

主要现象：

- 在51帧时，无 MLP 略好，说明可见信息足够时直接拟合可以很有效。
- 随着可见帧减少，无 MLP 误差快速增长，MLP从约45帧开始表现出更稳定的整体误差。
- 20帧时，MLP在8个独立轨迹组中的7个上更准确。
- 20帧组均误差为：MLP `17.416 cm`，无 MLP `54.195 cm`。
- MLP并非在所有窗口都占优：32个可见帧位置中，MLP组均误差较低15次，无 MLP较低17次。MLP的优势主要集中在可见信息较少的阶段。

## 30点状态窗口对比实验

在300 FPS下，30个采样点对应约100 ms的观测长度。实验使用与20点模型完全相同的训练、验证和测试文件划分，并重新训练30点状态窗口模型；无 MLP 方法仍使用相同可见点、参数约束和求根过程。

30点模型在第61轮早停，训练、验证和测试前缀数分别为366、94和64。测试前缀上的参数 MAE 为：`c1=26.176`、`c2=40.879`、`c3=1254.401`。

![30点状态窗口的落点误差对比](20260418_sliding_visibility_comparison_w30.png)

| 可见帧数 | 30点 MLP（cm） | 无 MLP（cm） | 20点 MLP（cm） |
| -------: | -------------: | -----------: | -------------: |
|       51 |     **10.405** |       11.448 |         12.887 |
|       45 |     **11.751** |       14.837 |         11.857 |
|       40 |     **12.683** |       17.445 |         13.045 |
|       35 |     **12.877** |       17.654 |         13.455 |
|       30 |     **15.395** |       26.897 |         14.891 |

结果表明：

- 在 `51→30` 的22个可见帧位置上，30点 MLP 的组均误差均低于无 MLP。
- 恰好只看30点时，30点 MLP 为 `15.395 cm`，无 MLP 为 `26.897 cm`；MLP在8个独立轨迹组中的7个上更准确。
- 在共同的 `51→30` 区间内，30点模型的平均组均误差为 `12.494 cm`，20点模型为 `13.283 cm`，30点模型整体更稳定。
- 但在30帧端点，20点模型为 `14.891 cm`，反而比30点模型低 `0.503 cm`。因此，增加状态窗口没有在每个位置都带来提升。

30点窗口能够使用更长的运动片段估计位置、速度和加速度，通常有助于抑制短窗口噪声；代价是训练前缀由478个减少为366个，而且更长窗口可能平滑掉局部变化。当前测试只有8个独立轨迹组，亚厘米级的20点与30点差异不足以证明30点必然更优，但30点 MLP 相比无 MLP 的优势在本次同划分实验中是稳定的。

## 30点MLP超参数搜索

为判断MLP是否还能通过常用超参数优化获得提升，本实验固定30点状态窗口和数据划分，搜索学习率、weight decay、dropout、batch size与隐藏层宽度。第一阶段比较36组配置，全部使用随机种子42；第二阶段将验证集前5名分别用随机种子42、43、44复验。配置选择只依据验证集恰好30点可见时的轨迹组均XY误差，开发测试集在最终配置确定后才参与评估。

![MLP超参数搜索结果](20260418_mlp_hyperparameter_search.png)

最终入选配置为：

| 超参数 | 取值 |
|---|---:|
| 隐藏层 | `256 -> 128 -> 64` |
| 学习率 | `0.002198` |
| Weight decay | `0.00006293` |
| Dropout | `0.07976` |
| Batch size | `128` |
| 种子42最佳轮次 | `9` |

原始30点配置在种子42下的验证误差为 `26.786 cm`；入选配置在相同种子下为 `18.581 cm`，降低约30.6%。入选配置的三个种子验证误差分别为 `18.581`、`22.168` 和 `17.371 cm`，平均为 `19.373 ± 2.037 cm`。这说明搜索确实找到了对当前验证集更有效、且在三个初始化下均能正常工作的配置。

![调参后30点模型的滑窗误差](20260418_sliding_visibility_comparison_w30_tuned.png)

| 可见帧数 | 调参后30点 MLP（cm） | 原30点 MLP（cm） | 原20点 MLP（cm） | 无 MLP（cm） |
|---:|---:|---:|---:|---:|
| 51 | 10.872 | **10.405** | 12.887 | 11.448 |
| 45 | 12.681 | **11.751** | 11.857 | 14.837 |
| 40 | 13.616 | **12.683** | 13.045 | 17.445 |
| 35 | 13.256 | **12.877** | 13.455 | 17.654 |
| 30 | 16.393 | 15.395 | **14.891** | 26.897 |

开发测试结果没有复现验证集上的提升：

- 30帧端点上，调参模型为 `16.393 cm`，比原30点模型高 `0.999 cm`，也高于原20点模型。
- 在共同的 `51→30` 区间内，调参模型平均误差为 `13.119 cm`，原30点模型为 `12.494 cm`。
- 调参模型只在22个可见帧位置中的3个优于原30点模型，但仍在全部22个位置上优于无 MLP。
- 30帧时，调参模型在8个独立轨迹组中的6个上优于无 MLP；原30点模型为7个。

因此，**本次主要超参数搜索没有提高开发测试集精度，当前不应使用调参模型替换原30点模型**。验证集改善但开发测试退化，较可能来自验证集只有8个独立轨迹组、搜索次数相对样本量较多，以及不同动作类型分布差异造成的验证集过拟合。调参模型与全部搜索记录仍被保留用于复现，但当前部署候选仍应选择原30点模型；若要继续调参，应增加轨迹数量并准备新的盲测集。

## 20帧时MLP更好的原因

20帧下降段对于同时估计全部指数参数而言较短。多组不同的 `a/b/c` 可以在窗口内产生相似曲线，但外推到地面时差异很大。特别是落地时间由高度方程的根决定，参数的小偏差会同时放大时间、X坐标和Y坐标误差。

MLP的作用是根据训练轨迹中学到的位置、速度和加速度关系，为 `c1/c2/c3` 提供随当前状态变化的先验。固定这三个参数后，每个轴需要从短窗口拟合的自由参数更少，因此短窗口外推更稳定。

这不是简单使用一个全局平均参数。20帧下的诊断结果为：

| 方法                              | 组均误差（cm） |
| --------------------------------- | -------------: |
| 正常训练的MLP                     |     **17.416** |
| 无 MLP、相同参数约束              |         54.195 |
| 固定使用训练集平均 `c`            |         56.351 |
| 打乱训练轨迹与 `c` 标签后重新训练 |         60.486 |

标签置乱后性能消失，固定平均参数也没有获得低误差，说明正式模型确实学习到了“当前运动状态 → 轨迹参数”的关系，而不是仅记住总体均值。

## 公平性结论

### 已确认不存在的直接泄露

- 测试轨迹及其落点没有进入训练样本。
- 同一轨迹的不同版本没有跨集合。
- 归一化和早停没有使用测试集。
- 测试时MLP只接收当前可见的20或30个点，不接收真实落点或未来点。
- 测试参数标签只用于计算参数误差，不参与落点推理。

### 已修正的不公平因素

早期对比中，无 MLP 使用的参数约束与MLP路径并不完全一致，20帧误差曾被高估为 `48.280 cm`。统一约束后，该批文件的对应误差下降，最终按独立轨迹组统计为 `54.195 cm`；数值变化还包含无偏划分和组级聚合的影响。

因此不能继续引用旧的 `9.077 cm vs 48.280 cm` 作为最终结论。当前有效结论是：

> 在先划分、测试集不筛选、参数约束一致并按独立轨迹组等权统计后，20帧下MLP为 `17.416 cm`，无 MLP为 `54.195 cm`。

### 仍然存在的评估限制

训练和测试虽然是不同轨迹，但都包含来自相同动作轮次的样本，例如 `cuoqiu_round01` 和 `diaoqiu_round01`。这不是标签泄露，但属于同场景、同分布测试，可能高估跨录制轮次、跨球员或跨标定环境的泛化能力。

此外，当前下降段数据的具体生成过程没有完整元数据，不能完全排除上游处理已经利用完整轨迹或落点信息。由于MLP和无 MLP使用相同测试输入，这对两者相对比较的影响较小，但会使绝对误差偏乐观。

所以可以确认：**当前代码路径中没有发现直接的训练测试标签泄露，且求解约束已经统一；但尚不能声称这是跨场景、完全外部独立的无泄露评估。**要得到更强结论，仍需按录制轮次整体留出，或者使用全新采集、从未参与开发的数据集。

## 与论文实验的主要差距

- 论文使用击球后早期轨迹；当前数据是下降阶段轨迹。
- 论文160 FPS下20点约为125 ms；当前按300 FPS解释时20点约为66.7 ms。
- 当前数据缺少真实逐点帧号和精确落地时间。
- 当前评价地面落点，而论文更关注机器人可执行击球点和击球时间。
- 当前测试只有8个独立轨迹组，远不足以代表全部动作和场景。

## 当前结论

经过重新划分、公平基线和泄露审计后，MLP在20帧下降段上的优势仍然存在，其主要来源是学习到的状态相关参数先验，而不是测试标签输入、全局参数均值或更宽松的求解条件。

30点超参数搜索改善了验证集，却未改善开发测试集，因此当前继续保留原30点模型，不用调参模型替换。

但这一结论只适用于当前同场景下降段数据。下一步应按完整录制轮次进行分组留出，并使用带真实击球帧、逐点帧号和落地帧的新数据做最终验证。
