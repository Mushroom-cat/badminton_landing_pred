# 20260418 轨迹落点预测对比实验报告

## 实验目标

实验比较两种方法：

1. **MLP 方法**：根据最近20或30个轨迹点的状态预测关键参数 `c1/c2/c3`，再使用当前全部可见点拟合其余参数并计算落点。
2. **无 MLP 方法**：直接从当前全部可见轨迹拟合全部参数并计算落点。

两种方法使用相同的测试轨迹、轨迹公式、参数约束和落点定义，主要指标为预测落点与真实落点的 XY 平面距离。

## 轨迹公式与预测流程

实验采用论文中的三轴轨迹形式：

```text
x(t) = a1 * exp(b1 * t) + c1
y(t) = a2 * exp(-0.002 * t) + b2 * t + c2
z(t) = a3 * exp(b3 * t) - 0.79 * t + c3
```

其中，`c1/c2/c3` 是三条坐标曲线中的关键偏置参数。MLP 并不直接输出落点，而是根据最近20或30个可见点提取的位置、速度和加速度，预测这三个参数；随后由当前全部可见点拟合剩余参数。无 MLP 方法则从相同的全部可见点直接拟合全部参数。

两种方法最后都通过求解 `z(t)=0` 得到落地时间，再代入 `x(t)` 和 `y(t)` 得到预测落点。因此，对比中的输入范围、轨迹方程、落地高度、求根过程和误差定义均保持一致，区别只在于 `c1/c2/c3` 是由 MLP 提供，还是随其他参数一起直接拟合。

当前有效实验口径是：**轨迹参数拟合始终使用当前全部可见帧**。20或30帧窗口只表示 MLP 状态特征的提取范围，不再表示轨迹参数拟合的数据范围。

## 数据划分与防泄露处理

数据集原始共有78个文件。划分严格在参数标签质量筛选之前进行，并按原始轨迹组完成；当前评估中已删除异常长版本测试文件 `cuoqiu_round01_017875_1.txt`，因此测试结果使用剩余8个测试文件。

| 集合       | 文件数 | 说明                                  |
| ---------- | -----: | ------------------------------------- |
| 原始训练组 |     60 | 其中4条参数标签退化，仅从训练组中排除 |
| 实际训练集 |     56 | 478个训练前缀                         |
| 验证集     |      9 | 112个验证前缀                         |
| 测试集     |      8 | 删除异常测试文件后，对应8个独立轨迹组 |

原测试集中有一条轨迹同时存在普通版本和较长的 `_1` 版本；当前已删除较长版本，统计总体误差时按8个独立轨迹组等权计算。

审计结果：

- 训练、验证、测试文件名无交集。
- 三个集合的轨迹组无交集。
- 训练与测试之间没有完全相同的轨迹内容。
- 训练与测试之间没有完全相同的真实落点。
- 输入与目标的归一化统计仅由训练集计算。
- 测试落点和测试参数标签不参与模型训练、早停或归一化。

## 20点MLP训练设置

- 输入：位置、速度、加速度，共9维
- 状态窗口：最近20个轨迹点
- 网络：`9 -> 128 -> 128 -> 64 -> 3`
- 输出：`c1/c2/c3`
- 激活函数：ReLU
- Dropout：0.1
- 损失函数：SmoothL1
- 优化器：AdamW
- 初始学习率：`1e-3`
- Weight decay：`1e-5`
- Batch size：64
- 最大训练轮数：300
- Early stopping patience：30
- 随机种子：42
- 实际早停：第123轮

测试前缀上的参数 MAE 为：

| 参数 |      MAE |
| ---- | -------: |
| `c1` |   30.380 |
| `c2` |   40.629 |
| `c3` | 1020.299 |

参数误差不等同于落点误差，因为后续拟合的其他参数可能产生补偿，最终仍以落点误差为主要指标。

## 前缀20帧时MLP更好的原因

20帧下降段对于同时估计全部指数参数而言较短。多组不同的 `a/b/c` 可以在少量可见点内产生相似曲线，但外推到地面时差异很大。特别是落地时间由高度方程的根决定，参数的小偏差会同时放大时间、X坐标和Y坐标误差。

MLP的作用是根据训练轨迹中学到的位置、速度和加速度关系，为 `c1/c2/c3` 提供随当前状态变化的先验。固定这三个参数后，每个轴需要从短可见轨迹中估计的自由参数更少，因此早期外推更稳定。

删除异常长版本测试样本后，前缀20帧下 MLP 组均误差为 `12.156 cm`，无 MLP 为 `53.423 cm`。这说明在短可见轨迹阶段，状态相关的 `c1/c2/c3` 先验能够显著降低外推不稳定性。

## 公平性结论

### 已确认不存在的直接泄露

- 测试轨迹及其落点没有进入训练样本。
- 同一轨迹的不同版本没有跨集合。
- 归一化和早停没有使用测试集。
- 测试时MLP只接收当前可见的20或30个点，不接收真实落点或未来点。
- 测试参数标签只用于计算参数误差，不参与落点推理。

### 已修正的不公平因素

早期对比中，无 MLP 使用的参数约束与MLP路径并不完全一致，因此不能继续引用旧的 `9.077 cm vs 48.280 cm` 作为最终结论。

当前有效结论是：

> 在先划分、删除异常长版本测试样本、参数约束一致并按独立轨迹组等权统计后，20帧下MLP为 `12.156 cm`，无 MLP为 `53.423 cm`。

### 仍然存在的评估限制

训练和测试虽然是不同轨迹，但都包含来自相同动作轮次的样本，例如 `cuoqiu_round01` 和 `diaoqiu_round01`。这不是标签泄露，但属于同场景、同分布测试，可能高估跨录制轮次、跨球员或跨标定环境的泛化能力。

此外，当前下降段数据的具体生成过程没有完整元数据，不能完全排除上游处理已经利用完整轨迹或落点信息。由于MLP和无 MLP使用相同测试输入，这对两者相对比较的影响较小，但会使绝对误差偏乐观。

所以可以确认：**当前代码路径中没有发现直接的训练测试标签泄露，且求解约束已经统一；但尚不能声称这是跨场景、完全外部独立的无泄露评估。**要得到更强结论，仍需按录制轮次整体留出，或者使用全新采集、从未参与开发的数据集。

## 与论文实验的主要差距

- 论文使用击球后早期轨迹；当前数据是下降阶段轨迹。
- 论文160 FPS下20点约为125 ms；当前按300 FPS解释时20点约为66.7 ms。
- 当前数据缺少真实逐点帧号和精确落地时间。
- 当前评价地面落点，而论文更关注机器人可执行击球点和击球时间。
- 当前测试只有8个独立轨迹组，远不足以代表全部动作和场景。

## 当前结论

经过重新划分、公平基线和泄露审计后，MLP在20帧下降段上的优势仍然存在，其主要来源是学习到的状态相关参数先验，而不是测试标签输入、全局参数均值或更宽松的求解条件。

但这一结论只适用于当前同场景下降段数据。下一步应按完整录制轮次进行分组留出，并使用带真实击球帧、逐点帧号和落地帧的新数据做最终验证。
